# 《孙子兵法》战略决策网站改进版 Plan.md

## 0. 文档说明

本文档是在原《孙子兵法 · 战略决策系统》网站方案基础上，根据《优缺点与改进.txt》中的问题反馈重新整理的改进版开发计划。

原网站的方向是正确的：它不是单纯展示《孙子兵法》原文，而是把十三篇转化为现代决策能力，例如“始计篇 = 胜算判断”“作战篇 = 成本控制”“谋攻篇 = 不战而胜”。同时，网站已经具备首页、十三篇、战略地图、概念库、案例库、决策训练、金句、学习路线、AI 辅助等完整产品结构。

但当前版本存在五个核心问题：

1. 内容太模板化，很多页面不像真正的逐句解读。
2. 十三篇原文不够完整，部分篇章更像节选。
3. 案例库过于泛化，缺少真实历史、商业、职场、技术细节。
4. 决策训练更像选择题库，不像完整训练系统。
5. 概念库解释偏短，缺少古代语境、现代迁移、适用边界和反面案例。

因此，新版目标不是简单“增加内容”，而是把网站从：

```text
概念型内容网站
```

升级为：

```text
完整原文校注 + 深度案例分析 + 多步骤决策训练 + 可复盘的战略学习系统
```

## 1. 改进版项目定位

### 1.1 项目名称

**孙子兵法 · 战略决策系统 Pro**

### 1.2 一句话定位

把《孙子兵法》做成一个可以逐句读懂、案例看透、场景练会、现实可用的战略思维训练网站。

### 1.3 产品核心变化

旧版重点：

```text
十三篇阅读 → 战略地图 → 案例库 → 决策训练
```

新版重点：

```text
完整原文 → 逐句译注 → 篇章结构 → 真实案例 → 多步训练 → 用户复盘 → AI 辅助校正
```

### 1.4 新版核心目标

网站要解决六个问题：

1. **读不全**：用户需要看到完整十三篇，而不是节选。
2. **读不准**：用户需要明确区分“白话翻译”和“现代总结”。
3. **读不深**：用户需要知道字词、句义、篇章结构、思想逻辑。
4. **看不透案例**：案例必须有真实人物、资源、约束、决策节点、结果。
5. **练不出能力**：训练不能只是选 A/B/C/D，而要形成判断流程。
6. **用不稳**：概念必须说明适用场景、不适用场景、误用风险。

## 2. 改进总原则

### 2.1 内容原则

所有《孙子兵法》内容必须区分四层：

```text
原文句子 → 字词解释 → 白话翻译 → 现代应用
```

禁止把“现代总结”伪装成“白话译文”。

示例：

| 层级 | 内容 |
|---|---|
| 原文 | 兵贵胜，不贵久 |
| 字词 | 贵：重视；胜：取得有效结果；久：长期拖延、持续消耗 |
| 白话翻译 | 用兵重在迅速取得有效胜利，不应陷入长期消耗。 |
| 现代应用 | 项目、竞争、谈判中，不要为了“坚持”而无限投入资源，要及时判断投入产出比。 |

### 2.2 完整性原则

每篇必须明确：

```text
当前文本版本：通行本
完整状态：完整 / 待校对 / 节选
异文校注：暂无 / 有
校对状态：未校对 / 初校 / 复校 / 已定稿
```

页面顶部必须显示版本说明，增强可信度。

### 2.3 案例原则

案例不能只讲概念，必须讲具体事实。

每个案例必须包含：

1. 案例背景
2. 关键人物 / 组织
3. 双方资源对比
4. 当时关键约束
5. 决策节点
6. 如果硬拼会怎样
7. 对应《孙子兵法》原文
8. 最终结果
9. 现实迁移
10. 给用户的练习题

### 2.4 训练原则

训练不能只是题库，而要变成流程。

每道训练题必须包含：

```text
判断局势 → 识别资源 → 判断对手 → 选择策略 → 系统评分 → 错误思维 → 兵法依据 → 用户复盘
```

### 2.5 概念原则

每个概念必须说明：

1. 一句话解释
2. 原文出处
3. 古代语境
4. 现代解释
5. 适用场景
6. 不适用场景
7. 常见误解
8. 正确使用方法
9. 反面案例
10. 自测问题

### 4.1 首页目标调整

旧首页主要展示模块入口。

```text
这个网站不是名句集合，而是完整学习系统。
```

### 4.2 首页新增模块

#### 4.2.1 当前内容可信度说明

在首页增加：

```text
当前内容版本：
- 原文版本：通行本
- 十三篇完整度：100%
- 逐句译注完成度：待展示进度
- 案例深度化完成度：待展示进度
- 训练系统完成度：待展示进度
```

作用：

让用户知道网站不是随意生成内容，而是有校对、补全、迭代过程。

#### 4.2.2 学习闭环展示

首页展示新版闭环：

```text
完整原文
   ↓
逐句译注
   ↓
概念理解
   ↓
真实案例
   ↓
多步训练
   ↓
个人复盘
```

#### 4.2.3 示例体验区

首页放一个完整示例：

```text
原文：兵贵胜，不贵久
字词：贵，重视；久，拖延消耗
白话：用兵重在迅速取得有效结果，不应长期拖耗
现代应用：项目延期、价格战、团队内耗都要警惕长期消耗
```

作用：

用户不需要进入详情页，就能理解新版网站的深度。

### 7.5 示例：官渡之战深度案例

#### 案例标题

```text
官渡之战：小势力如何攻击大势力的关键弱点？
```

#### 案例背景

袁绍与曹操在北方争霸。袁绍拥有更强的兵力、资源和声望，曹操则兵少粮少，长期对峙会陷入明显劣势。

#### 关键人物

| 人物 | 角色 | 优势 | 劣势 |
|---|---|---|---|
| 袁绍 | 北方强势军阀 | 兵力多、资源厚、声望高 | 决策迟疑、内部意见不一 |
| 曹操 | 劣势一方 | 执行力强、决断快、用人灵活 | 兵少、粮少、压力大 |
| 许攸 | 袁绍谋士，后投曹操 | 掌握袁军粮草信息 | 立场变化带来风险 |
| 淳于琼 | 袁军粮草守将 | 掌管乌巢粮草 | 防守疏忽 |

#### 双方资源对比

| 维度 | 袁绍 | 曹操 |
|---|---|---|
| 兵力 | 优势明显 | 明显劣势 |
| 粮草 | 储备较强，但依赖乌巢 | 粮草紧张 |
| 声望 | 四世三公，号召力强 | 政治地位上升但压力大 |
| 决策 | 内部意见复杂 | 中枢决断较快 |
| 信息 | 关键粮仓位置暴露 | 借许攸获得关键信息 |

#### 关键转折

```text
曹操没有选择继续正面消耗，而是袭击乌巢粮仓。
```

#### 对应兵法

```text
避实击虚
攻其所必救
兵贵胜，不贵久
```

#### 现实迁移

```text
小团队面对大公司时，不应直接拼预算、渠道和人力，而应寻找对方短时间内无法修补的关键薄弱点。例如细分用户、特殊场景、服务响应速度、老产品体验缺口。
```

## 8. 决策训练系统改进计划

### 8.1 当前问题

当前训练题数量足够，但存在三个问题：

1. 题目重复度高。
2. 训练流程太短，只是选择题。
3. 评分维度只是展示，没有完整解析体系。

### 8.2 改进目标

把训练系统从“选择题”升级为“战略推演训练”。

### 8.3 新版训练流程

每道题必须包含 8 步：

```text
第一步：判断局势
第二步：识别资源
第三步：判断对手强弱
```

### 8.6 训练交互要求

用户必须完成前置判断后，才能选择策略。

流程：

```text
先判断局势
再识别资源
再分析对手
最后选择策略
```

不能一进入页面就直接选答案。

### 8.7 每个选项必须有完整解释

示例：

```text
A. 正面降价竞争

得分：35/100

为什么可能有吸引力：
- 看起来可以快速抢市场。
- 对用户短期有刺激。

为什么风险大：
- 对方资源更强，更能承受低价。
- 小公司现金流更脆弱。
- 容易把竞争变成消耗战。

对应兵法：
兵贵胜，不贵久。

错误思维：
把“行动很激烈”误认为“战略很有效”。
```

### 8.8 最优策略必须说明前提

不允许只写：

```text
推荐选择：C
```

必须写：

```text
在以下前提下，C 是更优选择：
1. 你确实在细分用户中有口碑。
2. 大公司短期无法深入服务该群体。
3. 你的现金流不足以支持长期价格战。
4. 细分市场仍有足够商业价值。
```

禁止只改类别、难度、标题就重复使用同一题。

### 9.5 示例：道天地将法

#### 一句话解释

```text
“道天地将法”是行动前判断胜算的五个基本维度。
```

#### 维度拆解

| 维度 | 古代含义 | 现代迁移 |
|---|---|---|
| 道 | 上下同欲 | 组织共识、共同目标 |
| 天 | 时令气候 | 外部周期、时间窗口、政策环境 |
| 地 | 地形远近险易 | 市场位置、资源环境、渠道结构 |
| 将 | 将领能力 | 负责人能力、判断力、执行力 |
| 法 | 制度军纪 | 流程、规则、激励机制 |

#### 适用场景

1. 项目立项前。
2. 创业方向判断。
3. 团队重大变革。
4. 市场竞争策略制定。
5. 技术重构决策。

#### 不适用场景

1. 已经进入紧急执行阶段，无法完整评估时。
2. 信息极度不足，五个维度都无法判断时。
3. 只需要低成本试错的小决策。

#### 常见误解

```text
误解一：把“道”理解成道德口号。
正确理解：“道”更强调上下目标一致和组织共识。

误解二：只分析外部环境，不分析自己组织能力。
正确理解：五个维度必须合在一起看。
```

#### 正确使用方法

```text
不要只问“这个机会好不好”，而要问：
1. 我们是否目标一致？
2. 当前时机是否成熟？
3. 资源环境是否支持？
4. 负责人是否可靠？
5. 制度流程能否支撑执行？
```

## 15. 内容生产标准

### 15.1 篇章内容标准

每篇必须达到：

```text
完整原文：100%
逐句拆分：100%
字词解释：每句至少 1-5 个重点词
白话翻译：每句必须有
现代应用：每句或每段必须有
篇章结构图：必须有
核心思想总结：800-1500 字
现实案例：至少 3 个
训练题：至少 2 个
复盘模板：必须有
```

每个案例必须达到：

```text
背景：300-600 字
人物 / 组织：至少 2 个
资源对比：至少 5 个维度
关键约束：至少 3 条
时间线：至少 3 个节点
决策节点：至少 2 个
对应原文：至少 1 条
现实迁移：300-800 字
练习题：至少 3 题
```

每题必须达到：

```text
背景：300-800 字

策略选项：4-6 个
每个选项完整反馈：必须有
评分规则：7 个维度
兵法原文依据：至少 1 条
错误思维：至少 2 条
复盘问题：至少 5 个
```


### 19.1 内容验收

- 十三篇通行本完整原文全部上线
- 每篇都有版本说明
- 每篇都有篇章结构图
- 每句都有白话翻译
- 重点句都有字词解释
- 白话翻译与现代应用明确区分
- 至少 20 个真实深度案例
- 至少 30 个多步骤训练题
- 至少 20 个深度概念页
- 至少 30 条金句详情页

### 19.2 案例验收

每个案例必须有：

- 具体人物或组织
- 双方资源对比
- 关键约束
- 决策节点
- 兵法原文依据
- 最终结果
- 现实迁移
- 用户练习题

### 19.3 训练验收

每道训练题必须有：

- 局势判断
- 资源识别
- 对手分析
- 策略选择
- 评分
- 每个选项反馈
- 错误思维提醒
- 兵法原文依据
- 适用前提
- 用户复盘

### 19.4 概念验收

每个概念必须有：

- 古代语境
- 现代解释
- 适用场景
- 不适用场景
- 常见误解
- 正确使用方法
- 反面案例
- 自测问题

### 19.5 体验验收

- 用户进入首页 10 秒内能理解网站价值
- 用户 3 次点击内能进入任意篇章逐句学习
- 用户能从一句原文跳转到相关概念、案例、训练
- 用户完成训练后能得到具体反馈，而不是泛泛评价
- 用户能保存并回看复盘
- 移动端阅读体验流畅

### 19.6 AI 验收

- AI 回答必须引用站内原文或概念
- AI 能区分白话翻译和现代应用
- AI 能指出用户复盘中的错误思维
- AI 不鼓励攻击性、欺骗性、违法性行为
- 无 API Key 时能给出本地检索式回答

## 21. 最终产品形态

改进后的产品不再只是：

```text
《孙子兵法》内容展示网站
```

而应该成为：

```text
《孙子兵法》完整学习、深度理解、案例迁移、决策训练、个人复盘系统
```

最终用户路径应该是：

```text
看到一句原文
↓
理解每个关键词
↓
读懂白话意思
↓
知道现代场景怎么用
↓
看到真实案例如何发生
↓
进入训练题亲自判断
↓
得到系统反馈
↓
写下个人复盘
↓
形成自己的战略判断力
```

这个改进版的核心判断是：

> 网站的价值不在于“内容多”，而在于让用户从“知道一句兵法名言”变成“能在真实问题中判断局势、资源、成本、风险和时机”。

如果旧版解决的是“有没有完整产品结构”，新版要解决的是：

```text
内容是否可信
解释是否准确
案例是否真实
训练是否有效
用户是否真正形成能力
```
