# 《孙子兵法》战略决策网站 Plan.md

## 1. 项目名称

**孙子兵法 · 战略决策系统**

> 这不是一个单纯展示《孙子兵法》原文的网站，而是一个帮助用户读懂经典、理解战略、训练决策能力的系统型网站。

---

## 2. 项目定位

### 2.1 一句话定位

将《孙子兵法》从“古文阅读”升级为“战略思维训练系统”。

### 2.2 核心目标

网站要解决四个问题：

1. **看不懂**：古文晦涩，用户不知道每句话是什么意思。
2. **记不住**：十三篇内容分散，缺少结构化记忆方式。
3. **不会用**：用户知道“知己知彼”，但不知道如何应用到现实问题。
4. **缺训练**：只读文字很难形成战略判断力，需要案例与推演。

### 2.3 目标用户

| 用户类型 | 核心需求 |
|---|---|
| 普通读者 | 快速读懂《孙子兵法》 |
| 学生 | 学习古文、历史、思想内容 |
| 管理者 | 学习战略、组织、竞争、决策 |
| 创业者 | 判断机会、竞争格局、资源投入 |
| 产品经理 | 做市场分析、产品定位、差异化竞争 |
| 程序员 / 架构师 | 用战略思维理解系统设计、项目推进、技术债治理 |
| 军事 / 历史爱好者 | 通过战争案例理解兵法思想 |

## 3. 产品核心理念

《孙子兵法》不是鼓励战争，而是强调：

- 先判断能不能赢，再决定要不要行动。
- 最高级的胜利不是硬拼，而是以最小代价达成目标。
- 真正的胜利往往在开战前已经决定。
- 信息、时机、地形、组织、领导、制度都会影响结果。
- 战略不是口号，而是对成本、资源、风险、对手和局势的综合判断。

因此网站设计不能只做“原文 + 翻译”，而要形成完整闭环：

```text
原文阅读 → 逐句理解 → 思想结构 → 案例迁移 → 决策训练 → 个人复盘
```

## 6. 首页设计

### 6.1 首页目标

首页要完成三件事：

1. 告诉用户《孙子兵法》不是单纯的军事书，而是战略决策书。
2. 让用户快速进入十三篇、战略地图、案例库和训练系统。
3. 建立网站气质：古典、冷静、克制、深度、有战略感。

### 6.2 首页结构

#### 6.2.1 Hero 区域

标题：

```text
孙子兵法 · 战略决策系统
```

副标题：

```text
从古代兵法到现代决策，学习如何判断胜算、控制成本、寻找时机、避免硬拼。
```

核心引用：

```text
兵者，国之大事，死生之地，存亡之道，不可不察也。
```

按钮：

- 开始阅读十三篇
- 查看战略地图
- 进入决策训练

设计原因：

首页不能直接堆古文，否则用户没有进入动力。要先告诉用户：这个网站解决的是“如何做正确决策”的问题。

要求：

- 不使用过度文言化表达
- 尽量保持准确、自然、易懂
- 复杂句拆成短句

---

#### 7.2.4 关键词注释区

解释关键概念，例如：

| 词语 | 解释 | 现代对应 |
|---|---|---|
| 道 | 上下目标一致 | 愿景、共识、组织目标 |
| 天 | 时令、时机、外部环境 | 市场周期、政策环境、行业趋势 |
| 地 | 地理条件、空间远近、险易 | 资源条件、渠道、场景、市场结构 |
| 将 | 将领能力 | 领导力、判断力、执行力 |
| 法 | 制度、纪律、组织方式 | 流程、制度、分工、激励机制 |

### 10.4 示例案例：大型系统重构

标题：

```text
大型系统重构：为什么不能一上来全面推翻旧系统？
```

对应思想：

- 始计篇：先计算胜算
- 作战篇：控制长期消耗
- 军形篇：先胜后战
- 虚实篇：避实击虚

分析：

```text
大型系统重构最容易失败的原因，是低估了业务依赖、历史逻辑、人员协作和上线风险。
如果一开始就全面推翻旧系统，相当于“先战后求胜”。
更好的方式是先找到风险较小、收益明确、边界清晰的模块做局部突破。
通过局部成功建立信任、积累工具、验证架构，再逐步扩大范围。
这符合“胜兵先胜而后求战”。
```

### 11.4 评分维度

| 维度 | 说明 |
|---|---|

| 成本 | 是否考虑资源消耗 |
| 风险 | 是否识别失败后果 |
| 时机 | 是否选择合适时间点 |
| 信息 | 是否掌握足够事实 |
| 对手 | 是否理解竞争方弱点 |
| 执行 | 策略是否可落地 |

---

### 11.5 示例训练题

```text
场景：
你是一家小公司的负责人。行业内一家大公司即将进入你的市场。
你资源有限，但你在一个细分用户群体中有较强口碑。
你应该如何行动？

选项：
A. 正面降价竞争
B. 大量投广告抢市场
C. 深耕细分用户，避开正面战场
D. 模仿大公司的产品线
E. 暂停业务，等待市场变化
```

反馈：

```text
推荐选择：C
对应思想：避实击虚、不战而屈人之兵
解析：
面对资源明显强于自己的对手，不宜进入对方擅长的主战场。
应寻找对方短时间内无法覆盖、但自己有优势的细分场景。
这不是逃避竞争，而是重新选择战场。
```

## 14. AI 辅助解读系统

### 14.2 功能

- 先引用相关原文
- 再解释含义
- 再说明现代应用
- 最后给出风险提醒
- 不鼓励攻击性、伤害性、违法性行为
- 强调《孙子兵法》的重点是减少误判和降低代价

## 16. 技术方案

### 16.1 前端技术栈

推荐：

```text
Next.js + TypeScript + Tailwind CSS + MDX + Zustand + Framer Motion
```

原因：

- Next.js 适合内容型网站和 SEO。
- TypeScript 保证数据结构清晰。
- Tailwind CSS 适合快速构建统一视觉系统。
- MDX 适合管理经典文本、注释和长文章。
- Zustand 适合管理阅读进度、收藏、训练状态。
- Framer Motion 可用于轻量过渡动画。

### 16.2 后端技术栈

MVP 阶段可以先使用静态数据：

```text
/content/chapters/*.mdx
/content/concepts/*.mdx
/content/cases/*.mdx
/content/training/*.json
```

进阶阶段：

```text
Next.js API Routes / NestJS + PostgreSQL + Prisma
```

### 16.3 搜索方案

MVP：

```text
本地 Fuse.js 搜索
```

进阶：

```text
Meilisearch / Typesense
```

搜索范围：

- 原文
- 译文
- 注释
- 概念
- 案例
- 金句

### 16.4 AI 功能方案

```text
前端输入问题 → API Route → 大模型接口 → 检索相关原文 / 概念 / 案例 → 生成回答
```

要求：

- 使用 RAG 检索站内内容
- 回答必须引用相关篇章

## 28. 最终判断

这个网站的核心不是“展示《孙子兵法》”，而是让用户完成一次认知升级：

```text
从背诵名句
到理解体系
到分析案例
到训练决策
到应用现实
```

最终产品形态应该是：

> 一个以《孙子兵法》为内容基础，以战略地图为知识结构，以案例库为现实连接，以决策训练为核心体验的战略思维学习网站。

如果只做阅读器，它只是一个经典文本网站。

如果做成战略决策系统，它才真正体现《孙子兵法》的现代价值。
